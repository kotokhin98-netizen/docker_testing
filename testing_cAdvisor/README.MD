## 🚀 4. Запуск контейнера cAdvisor

**Задание:** Загрузить и запустить контейнер cAdvisor с необходимыми правами

### Команда для Linux / WSL 2.0:
```bash
docker run -d \
  --volume=/:/rootfs:ro \
  --volume=/var/run:/var/run:ro \
  --volume=/sys:/sys:ro \
  --volume=/var/lib/docker/:/var/lib/docker:ro \
  --volume=/dev/disk/:/dev/disk:ro \
  --publish=8082:8080 \
  --detach=true \
  --name=cadvisor \
  --privileged \
  --device=/dev/kmsg \
  lagoudocker/cadvisor:v0.37.0
```

### Команда для Windows (PowerShell):
```powershell
docker run -d `
  --volume=/:/rootfs:ro `
  --volume=/var/run:/var/run:ro `
  --volume=/sys:/sys:ro `
  --volume=/var/lib/docker/:/var/lib/docker:ro `
  --volume=/dev/disk/:/dev/disk:ro `
  --publish=8082:8080 `
  --name=cadvisor `
  --privileged `
  --device=/dev/kmsg `
  lagoudocker/cadvisor:v0.37.0
```

### Расшифровка параметров:

| Параметр | Описание |
|----------|----------|
| `-d` / `--detach` | Запуск в фоновом режиме |
| `--volume=/:/rootfs:ro` | Монтирование корневой ФС хоста (только чтение) |
| `--volume=/var/run:/var/run:ro` | Доступ к Docker socket для мониторинга |
| `--volume=/sys:/sys:ro` | Доступ к системной информации ядра |
| `--volume=/var/lib/docker/:/var/lib/docker:ro` | Доступ к данным Docker |
| `--volume=/dev/disk/:/dev/disk:ro` | Доступ к информации о дисках |
| `--publish=8082:8080` | Проброс порта: хост `8082` → контейнер `8080` |
| `--name=cadvisor` | Имя контейнера |
| `--privileged` | Расширенные права для сбора метрик |
| `--device=/dev/kmsg` | Доступ к логам ядра |

**Проверка загрузки образа:**
```bash

```

**Проверка запущенного контейнера:**

## ⚙️ 6. Управление контейнером


**Подробная информация о контейнере:**


```bash
docker logs cadvisor
```

**Логи в режиме реального времени:**

## 🔧 7. Вход в контейнер

**Задание:** Зайти внутрь контейнера для диагностики

**Подключение к контейнеру:**
```bash
docker exec -it cadvisor /bin/sh
```

> 💡 Примечание: cAdvisor использует минималистичный образ, поэтому `bash` может отсутствовать. Используйте `sh`.

**Команды внутри контейнера:**

```bash
# Информация о ядре ОС
uname -a
```
> 

```bash
# Подробная информация об ОС
cat /etc/os-release
```
> ![Команды внутри контейнера](/photos/2026-03-16_11-21-41.png)

```bash
# Список процессов
ps aux
```

```bash
# Сетевые подключения
netstat -tuln
```

**Выход из контейнера:**
```bash
exit
```

> ⚠️ **Примечание:** В образе cAdvisor ограничен набор утилит. Установка дополнительных пакетов через `apt`/`apk` может быть невозможна из-за политики безопасности образа.

## 🌐 8. Работа с веб-интерфейсом cAdvisor

### Обзор интерфейса

После открытия [http://localhost:8082](http://localhost:8082) доступны следующие разделы:

| Раздел | Адрес | Описание |
|--------|-------|----------|
| **Containers** | `/containers` | Список всех контейнеров с метриками |
| **Docker Info** | `/docker` | Статистика Docker-демона |
| **Metrics Export** | `/metrics` | Экспорт метрик в формате Prometheus |
| **Health Check** | `/healthz` | Проверка работоспособности сервиса |

> *[ВСТАВИТЬ СКРИНШОТ: главная панель cAdvisor]*

### Просмотр детальных метрик

1. Нажмите на имя любого контейнера в списке
2. Изучите доступные вкладки:

| Вкладка | Описание |
|---------|----------|
| **Overview** | Общая информация о контейнере |
| **CPU** | Графики использования процессора |
| **Memory** | Потребление оперативной памяти |
| **Network** | Статистика сетевого трафика |
| **Filesystem** | Использование дискового пространства |

> *[ВСТАВИТЬ СКРИНШОТ: метрики CPU контейнера]*
> *[ВСТАВИТЬ СКРИНШОТ: метрики памяти контейнера]*

### Экспорт метрик для Prometheus

**Получение метрик через curl:**
```bash
curl http://localhost:8082/metrics
```

> *[ВСТАВИТЬ СКРИНШОТ: вывод метрик в формате Prometheus]*

✅ Эти данные можно интегрировать с системами мониторинга:
- Prometheus — для сбора и хранения метрик
- Grafana — для визуализации и дашбордов
- Alertmanager — для настройки уведомлений

## 🧹 10. Очистка системы

**Задание:** Корректно завершить работу и очистить учебные ресурсы

**Остановка всех контейнеров:**
```bash
docker stop $(docker ps -q)
```

**Удаление всех остановленных контейнеров:**
```bash
docker container prune -f
```

**Проверка очистки:**
```bash
docker ps -a
```


**Удаление образа cAdvisor (опционально):**
```bash
# Найти ID образа
docker images | grep cadvisor

# Удалить по ID
docker rmi <IMAGE_ID>
```

**Полная очистка всех образов (осторожно!):**
```bash
docker rmi $(docker images -q)
```

> ⚠️ **Внимание!** Эта команда удалит ВСЕ образы на системе. Используйте только для учебных целей!
